1.项目背景

假如我想用win或者linux做一个代码评分器，首先数据库里面有一些代码题，然后你要写一个代码，有一些版本，其中版本是直接让大模型来评价你的代码是否合理，只不过这个大模型是特化来评价代码的，它会比通用型llm在判代码上更科学，比如说它内化了一些评价标准之类的。

所以我现在要开发‘代码评分器’的第一阶段：大模型 API 评测版。

2.其他llm的建议

我的网页版LLM建议我的项目采用 Python + FastAPI。你可以接受这个建议，也可以自己想出自己的一个业务方案。这取决于你自己。

3.plan mode下的具体任务

请不要写具体的业务代码，只进入 Plan 模式（已经在plan模式中）帮我规划以下三件事：

****数据流与接口设计****：学生提交代码、请求大模型、返回评分 JSON 的完整链路。

****System Prompt 框架设计****：如何设计系统提示词，能让大模型稳定、不跑偏地按照‘可读性 20% + 逻辑正确性 50% + 规范性 30%’的比例输出标准 JSON，防止幻觉？

****项目目录结构****：给出一个干净的 FastAPI 项目骨架，并告诉我需要安装哪些核心依赖（如 langchain 或直接用官方 SDK）。”

4\. 参考的架构：

我决定第一阶段参考 GitHub 上流行的 LLM-as-a-Judge 思路。请在 Plan 模式下，帮我设计一个能够稳定对代码进行多维度（规范、逻辑、性能）打分并返回结构化 JSON 的 System Prompt 框架。

还有，如果我的prompt有自相矛盾的地方，请根据自己的意见选取自己想要的方案。
